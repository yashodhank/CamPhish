## Findings considered and rejected

- **Soft-delete for data** (D-4): Not worth doing because hard-delete is appropriate for red-team tooling — data should be permanently deletable on demand. Adds complexity without proportional benefit.
- **file_type detection from base64** (S-6): The image is always stored as `.png` and served with `image/png` content type regardless of source format. Browsers handle this correctly. Fix would add complexity for no user-visible benefit.
- **WebRTC local IP reliability** (F-5): The race condition in ICE candidate collection is a well-known browser limitation. A robust fix would require significant refactoring of the fingerprint module for marginal gain in a deprecated API (some browsers already block `getUserMedia` without HTTPS).
